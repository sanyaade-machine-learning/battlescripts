BattleScripts is a site where you code game-playing algorithms using javascript, the most popular programming language in the world. Then you run your player against the code written by others and see who wins. The goal is to write the best player algorithm for a given game, and beat all the other opponents.

A Quick Overview Of How It Works

  • Pick a game you want to play - anything from Tic-Tac-Toe to Battle Tanks and more.
  • Use our built-in editor to code a "player" algorithm in javascript that can play the game.
    • Use our default template for a quick start
    • Or if you're an expert, code from scratch
  • Our game controller engine runs your player against an opponent and displays the game as it runs
    • The game controller calls each player's code when it is its turn
    • A JSON object describing the game's state is passed in
    • The player inspects the game state and makes its next move
    • Repeat until one player wins!
    • You make changes using the editor and debugging tools until your player is the smartest you can make it!
    • Some games may be simple enough to allow you to code an optimal player (like Tic-Tac-Toe)
    • Other games will never be solvable, so you write the best player you can

    FAQs

    How exactly do I code a Player?
    You create a javascript Player object that implements a couple API methods. Those methods get automatically called by the game controller. As long as you follow the simple API, your player will work. Don't worry, we provide you with a default Player object to get you started.
    Do I have to know how to program?
    Yes, or at least have a desire to learn. This is a site for people who can code and think through algorithms.
    Do I have to be an expert programmer?
    No! There are games at many skill levels, and simple players can be written with a minimal amount of javascript knowledge. BattleScripts can be a learning tool! Of course, for more complicated games, experienced programmers will have a significant advantage, and understanding javascript in depth will certainly help you to create more robust players.
    How does my player know what to do?
    Each time it's your player's turn, its move() method will be called and passed a JSON object. This object contains all the information about the game state that you need to know to make an informed decision about your next move. For most games, players take turns. In some games, players may move simultaneously. It depends on the game.
    Is there human intervention? Can I change the code between moves?
    No, the players are completely autonomous. They must know how to make a move in any situation, without any external input. For games like Tic-Tac-Toe, this is pretty simple. For games like Chess, not so much.
    Where is my player stored?
    All player code is stored in our database. You edit your code using our IDE, and save it while you work on it. Once it's ready, you PUBLISH it and make it visible to others to play against.
    Can I revise my player's code?
    Yes! You can continually work on your player and improve its algorithm. Whenever you RE-PUBLISH your player's code, it will be visible to others and they can play against you.
